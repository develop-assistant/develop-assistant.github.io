方案总结

服务间调用灰度方案(Consumer -> Provider)

  • P1、P2、C1、C2服务注册在nacos，并分别在nacos元数据配置版本号V1和V2
  • C用RestTemplate通过服务发现调用P
  • 自定义负载均衡策略
  • C读取自己元数据中的版本号, 代表自己想访问哪个版本的P(负载均衡策略)
  • 获取所有的P实例, 并过滤出符合自己版本的实例(负载均衡策略)
  • 最终的结果是C2调用P2, C1调用P1
